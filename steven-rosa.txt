    1. What is Semantic HTML?
        Semantic HTML is the use of semantic tags to better structure and describe an HTML file for readability.

    2. What is HTML used for? 
        HTML is used to provide the basic structure to display webpages on browsers.

    3. What is an attribute and where do we put it? 
        Attributes are additional information added to the start tag of an element. 

    4. What is the h1 tag used for? How many times should I use it on a page?
        h1 tag is a heading tag, and usually the title/most important heading on the file, and therefore should only be used once.

    5. Name two tags that have required attributes
        The <img> and <a> tags.

    6. What do we put in the head of our HTML document? 
        <head> is a container for the metadeta of our HTML document. The title of the webpage, style, and other such things are set there.

    7. What is an id? 
        an id is an attribute that gives that element a unique id.

    8. What elements can I add an id to? 
        Any elements

    9. How many times can I use the same id on a page? 
        Once

    10. What is a class? 
        a class is an attribute that assigns a class to an element, which is like a group.

    11. What elements can I add a class to? 
        Any elements.

    12. How many times can I use the same class on a page? 
        As many times as you want

    13. How do I get my link to open in a new tab?
        add the attribute target="_blank" to the <a> element. e.g <a href="www.link.com" target="_blank">

    14. What is the alt attribute in the image tag used for? 
        It is there in the event that the image cannot be displayed

    15. How do I reference an id?
        id is referenced using # . #id { }

    16. What is the difference between a section and a div
        A div is non semantic html, a section is semantic html

    17. What is CSS used for? 
        CSS is used to style an HTML file

    18. How to we select an element? Example - every h2 on the page
        h2 { }

    19. What is the difference between a class and an id? - Give me an example of when I might use each one
        a class can be used on multiple elements, such as multiple <p> elements. An id is unique to 1 element, such as on a singular <h2> element, or <body>, used to target a specific element.

    20. How do we select classes in CSS?
        .class { }

    21. How do we select a p element with a single class of “human””?
        p.human { }

    22. What is a parent child selector? When would this be useful?
        Parent child selectors are patterns used to select specific elements. This is useful in order to apply css styles to certain elements, without affecting others.

    23. How do you select all links within a div with the class of sidebar?
        div a.sidebar{ }

    24. What is a pseudo selector?
        psuedo selectors are keywords that are added to selectors to define a special state, such as :hover .

    25. What do we use to change the spacing between lines?
        line-height: 0px;

    26. What do we use to change the spacing between letters?
        letter-spacing: 0px;

    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
        text-transform: uppercase, lowercase, capitalize 

    28. How do I add a 1px border around my div that is dotted and black?
        div {
            border: 1px dotted black;
        }

    29. How do I select everything on the page? 
        * { }

    30. How do I write a comment in CSS?
         /*  ...  */

    31. How do I find out what file I am in, when I am using the command line? 
        pwd command

    32. Using the command line - how do I see a list of files/folders in my current folder?
        ls 

    33. How do I remove a file via the command line? Why do I have to be careful with this? 
        using the rm command. It is dangerous because those files are irreversibly deleted, and small typos can be problematic.

    34. Why should I use version control? 
        It's a non destructive workflow, if at any point you wish to go back to a previous change, you are able to do so safely.

    35. How often should I commit to github?
        Whenever you finish a step in your project, so often. Steps such as creating a class, or finishing basic html structure.

    36. What is the command we would use to push our repo up to github? 
        git push -u origin <branch> 
    37. Walk me through Lambda's git flow.
        You clone a repo, $cd to the directory, $git checkout -b <firstname-lastname>, make changes to code, $git status to double check changed files. $git add <files> to add files to the staging area, $git commit -m "Descriptive Message" to commit changes, $git push -u origin <firstname-lastname> to push changes to the remote repo, then we create a pull request.
